# Douyin Batch Download & Transcribe — 技术栈与设计思路

## 一、需求概述

**输入**：一个或多个抖音分享链接（可从分享文本中自动提取）
**输出**：每条视频的结构化信息，包含平台名、发帖人、发帖时间、视频标题、视频描述、视频文案（语音转文字）

---

## 二、整体技术栈

```
┌────────────────────────────────────────────┐
│                 输入层                       │
│  一个或多个抖音 URL 或分享文本              │
└──────────────────┬─────────────────────────┘
                   ▼
┌────────────────────────────────────────────┐
│         Playwright 浏览器拦截               │
│                                            │
│  真实 Chromium 会话访问抖音页面              │
│  拦截 /aweme/v1/web/aweme/detail/ 响应      │
│    → 提取元数据（标题/作者/时间/统计）       │
│  拦截 douyinvod.com CDN 响应                │
│    → 获取视频下载 URL                       │
│                                            │
│  特点：真实浏览器环境，无需 API Key          │
└──────────────────┬─────────────────────────┘
                   ▼
┌────────────────────────────────────────────┐
│            并行 MP4 下载                     │
│                                            │
│  通过 CDN URL 直接下载 MP4 文件              │

│  失败自动重试（指数退避）                    │

                   ▼

│          ffmpeg 音频提取                     │
│                                            │
│  从 MP4 提取音频                             │

│  （Whisper 输入格式）                        │
└──────────────────┬─────────────────────────┘
                   ▼
┌────────────────────────────────────────────┐
│            faster-whisper (本地 ASR)         │
│                                            │
│   将音频转写为文字（带时间戳）               │
│   串行处理，保证 GPU 安全                    │
│   模型加载一次，整个批次复用                 │
│                                            │

│  输出 JSON + TXT                               │
└────────────────────────────────────────────┘
```

## 三、核心选型理由

### 3.1 视频下载：Playwright 浏览器拦截

**为何选它**：

- 真实浏览器环境，与用户手动访问行为一致
- 通过拦截 Douyin detail API 和 CDN 响应获取视频 URL
- 无需逆向 API 签名算法，维护成本低
- 支持验证码检测，可切换有头模式处理

**不选的其他方案**：

| 方案 | 放弃原因 |
|------|---------|
| yt-dlp | 抖音反爬频繁，yt-dlp 的抖音提取器更新滞后 |
| Douyin_TikTok_Download_API | 依赖第三方在线解析服务，不稳定 |
| 手动解析 API | 需要逆向 X-Bogus 签名，维护成本高 |

### 3.2 语音转文字：faster-whisper（本地）

**为何只选本地**：

- 完全免费，无 API 费用
- 数据不上传，隐私安全
- 不依赖网络，离线可用
- 有 GPU 时 1h 音频 ≈ 40s，纯 CPU 也只要约 12min

**不选的其他方案**：

| 方案 | 放弃原因 |
|------|---------|
| Groq Whisper API | 用户要求纯本地，不依赖云端 |
| OpenAI Whisper API | $0.36/h，付费且依赖网络 |
| 阿里云 ASR | 付费、需国内认证、依赖网络 |
| 原版 whisper (openai-whisper) | 比 faster-whisper 慢 4-6 倍，内存高 |

### 3.3 简繁转换：OpenCC

**为何加入**：

- Systran faster-whisper 模型训练语料以繁体中文为主，`--language zh` 时输出繁体
- `opencc` 是中文简繁转换的事实标准，通用且零配置
- 可开关（`--no-simplify`），保持灵活性

## 四、并发模型

```
URL_1:  [解析]──[下载]──[提取音频]──[转写]──[输出]──[清理]
URL_2:  [解析]──[下载]──[提取音频]──[转写]──[输出]──[清理]
URL_3:      [解析]──[下载]──[提取音频]──[转写]──[输出]──[清理]
...
```

三层并发控制（基于计数信号量）：

| 阶段 | 并发数 | 原因 |
|---|---|---|
| 浏览器解析 | 3 | 吃内存，保持低 |
| MP4 下载 | 6 | 网络 IO，可以高 |
| Whisper 转写 | 1 | GPU 安全，串行 |

**自然流水线**：每个视频独立走完全流程。URL_1 下载完开始转写时，URL_2/URL_3 还在下载。Semaphore 自动协调，无需额外同步逻辑。

## 六、详细设计

### 6.1 浏览器拦截机制

Playwright 启动 Chromium 访问抖音页面，注册 response 事件监听：

1. **Detail API 拦截**：`/aweme/v1/web/aweme/detail/` 响应 → 解析 JSON 提取元数据和媒体 URL
2. **CDN 响应拦截**：`douyinvod.com` 响应 → 记录视频下载 URL
3. **候选排序**：按分辨率和文件大小排序，选择最优质量

### 6.2 转写子进程

Node.js 主进程通过 `spawn` 调用 Python 转写脚本：

```
Node:  spawn("python", ["transcribe.py"])
       ───stdin──→  {"wav_path": "...", "model": "small", ...}
       ←──stdout─── {"segments": [...], "transcript": "...", "meta": {...}}
```

Python 脚本只做转写（faster-whisper + OpenCC），纯函数，输入 WAV 路径，输出 JSON。

### 6.3 断点续传

`download-state.json` 记录每个 URL 的处理状态：
- `parsing`：正在解析
- `downloading`：正在下载
- `transcribing`：正在转写
- `completed`：完成
- `retrying`：重试中
- `permanent_failure`：永久失败
- `failed`：失败

重跑时自动跳过 `completed` 状态的项。

### 6.4 输出文件

每个视频输出 JSON（元数据 + 文案 + 分段时间戳）和 TXT（纯文本转写），MP4 和 WAV 文件保留在原位。
